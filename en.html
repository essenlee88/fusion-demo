Fusion Mode — Multi-Chain Entry Component Options

Meme Rush will support BSC / SOL / Base multi-chain trading on a single screen (Fusion Mode).
Users switch which chains to view/trade via a "chain selector" on the page.
There are two entry component options to choose from. Please review the comparison and share your preference.

PRD: Fusion Multi-Chain Trading PM: Elsa KO: 2026-03-25 Status: Pending scheduling, component option to be confirmed

Fusion Mode Overview

With Fusion enabled, the page switches from a single-chain view to a multi-chain mixed view. Three lists (New / Finalizing / Migrated) display tokens from multiple chains simultaneously, and My Positions on the right aggregates cross-chain holdings.

Meme Rush page with Fusion Mode enabled (BSC + SOL + Base mixed view)

Chain Selector Entry Points

There are 3 entry points on the page to open the chain selector. They are globally synced — switching at any one instantly updates the others.

1 Next to the Meme Rush tab
2 Next to My Position Bar
3 Top of My Position Widget

Option A vs Option B

The core difference: Single-select (Radio) vs Multi-select (Checkbox). The selector UI and list filtering logic differ between the two options.

A
Single Select Radio
InteractionSelect one chain, or select "BSC/SOL/BASE" for all-chain mode
Selection✓ marks current selection (mutually exclusive)
ListSingle chain → shows only that chain's tokens; All chains → mixed view
QuickBuySingle chain: 1 row / All chains: 3 rows (per-chain amount)

+ Simple and intuitive, consistent with current chain-switching UX

+ No edge case of "2 chains selected"

- Cannot flexibly combine (e.g. view BSC+SOL only, exclude Base)

B
Multi Select Checkbox
InteractionCheckbox to select any combination of chains (1/2/3)
Selection☑ multi-select, at least 1 chain must remain checked
ListDisplays tokens from whichever chains are checked
QuickBuyExpands input rows based on number of checked chains

+ Maximum flexibility, supports any 2-chain combination

+ Trigger directly shows selected chain icons

- More complex concept, users need to understand multi-select

- Sorting logic for 2-chain mixed view needs extra definition
