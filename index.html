页面上有 3 个入口可以打开链选择器，它们全局同步——任意一个切换，其他两个跟着变。

1 Meme Rush 标签旁
2 My Position Bar 旁
3 My Position Widget 顶部

方案 A vs 方案 B

两种方案的核心区别：单选（Radio）还是 多选（Checkbox）。点击入口后弹出的选择器 UI 不同，列表筛选逻辑也不同。

A
单选切换 Radio
交互选一条链，或选 "BSC/SOL/BASE" 全链模式
选中态✓ 标记当前选中（单选互斥）
列表单链 → 只显示该链 token；全链 → 三链混排
QuickBuy单链 1 行 / 全链 3 行（per-chain 金额）

+ 简单直觉，和当前切链体验一致

+ 没有「选了 2 条链」的边界 case

- 无法灵活组合（如只看 BSC+SOL，不看 Base）

B
多选组合 Checkbox
交互Checkbox 勾选任意链的组合（1/2/3 条）
选中态☑ 多选，至少保留 1 条链
列表勾选哪些链，就展示哪些链的 token
QuickBuy按勾选链数量展开 input

+ 灵活度最高，支持任意 2 链组合

+ 触发器直接展示已选链 icon

- 概念更复杂，用户需要理解多选

- 2 链混排时排序逻辑需额外定义
